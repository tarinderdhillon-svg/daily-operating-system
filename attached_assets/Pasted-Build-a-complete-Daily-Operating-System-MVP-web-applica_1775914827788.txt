3. API Endpoints:

   GET /
   - Serves the frontend HTML/CSS/JS
   - Main application page

   GET /health
   - Returns: {"status": "ok", "message": "Daily OS MVP running"}

   POST /api/create-task
   - Input: {title, due_date (optional), priority (optional: "High"/"Medium"/"Low")}
   - Creates page in Notion Tasks database
   - Sets properties:
     * Name: title
     * Due Date: due_date
     * Priority: [priority as multi_select]
     * Status: "Not started"
   - Output: {success: true, message: "✅ Task created", task_id: "..."}
   - Errors: {success: false, error: "description"}

   GET /api/get-tasks
   - Returns all tasks from Notion database
   - Categorizes into:
     * overdue: due_date < today
     * outstanding: today <= due_date <= today + 7 days
     * todo: due_date > today + 7 days OR no due_date
   - Output: {
       success: true,
       tasks: [...all task details...],
       categorized: {
         overdue: [...],
         outstanding: [...],
         todo: [...]
       }
     }

   POST /api/update-task
   - Input: {task_id, title (optional), status (optional), priority (optional), due_date (optional)}
   - Updates task in Notion
   - Output: {success: true, message: "✅ Task updated"}

   POST /api/delete-task
   - Input: {task_id}
   - Archives task in Notion (sets archived: true)
   - Output: {success: true, message: "✅ Task deleted"}

   POST /api/process-chat
   - Input: {message: "user input"}
   - Parse and process natural language commands:
     * Detect task creation: extract title, due_date, priority
     * Detect task updates: identify which task, what changed
     * Detect calendar queries: check /api/get-tasks for availability
     * Detect briefing requests: call /api/generate-briefing
   - Route to appropriate handler
   - Output: {
       success: true,
       response: "bot message to display",
       action_taken: "description",
       data: {...relevant data...}
     }

   POST /api/generate-briefing
   - No input required (or optional date parameter)
   - Fetch latest news from:
     * AI & Technology sources (TechCrunch, ArXiv, The Verge, etc.)
     * Business & Markets sources (Financial Times, Bloomberg, McKinsey, etc.)
   - Generate 6 articles per category
   - For each article:
     * Title
     * Publication source
     * Date
     * Enriched summary (2-3 sentences with business impact)
     * Key data points or metrics
     * Source link
   - Output: {
       success: true,
       briefing: {
         generated_at: "timestamp",
         ai_tech: [...articles...],
         business_markets: [...articles...]
       }
     }

   GET /api/get-calendar
